# food-recipy 업그레이드 검토 (2026-07-13)

현재: 레시피 **107종** · 6개국 · 한/일 이중언어 · 냉장고 찾기 · 통합 장보기 · 영양정보 · 메모/별점 · 쿡모드 · 요리별 아이콘 · 카테고리 하위분류 필터 · 조리단계 인라인 '🔎 왜?' 원리. 바닐라 정적앱 + GitHub Pages + PWA.

아래는 **다음 업그레이드 후보**를 체감 효과 대비 노력으로 우선순위화한 것.

---

## Tier A — 퀵윈 (노력 낮음 · 효과 큼) ★추천

### A1. 오프라인 완성도 (실질 결함 수정)
- **현상**: 서비스워커 SHELL이 코드/아이콘만 프리캐시하고 **레시피 JSON(data/recipes/*.json)은 캐시 안 함**. 한 번도 온라인에서 안 연 레시피는 오프라인(지하철·매장)에서 못 열 수 있음.
- **개선**: install 시 카테고리 JSON 전부 프리캐시 + `manifest.webmanifest`. 배포 스크립트로 파일 목록 자동 생성해 SW에 주입(누락 방지).
- 노력: 낮음 / 리스크: 캐시 용량 소폭↑(무시 가능).

### A2. 레시피 공유 버튼
- **왜**: "친구와 같이 본다"가 핵심 사용처. 지금은 특정 레시피를 콕 집어 보내기 어려움.
- **개선**: 상세화면에 공유 버튼 → Web Share API(모바일 카톡·라인 공유) + 폴백으로 딥링크(`#/recipe/…`) 클립보드 복사. 라우팅이 이미 딥링크 지원.
- 노력: 낮음.

### A3. 다크모드 수동 토글 + 글자 크기(쿡모드)
- **현상**: OS 설정 자동(prefers-color-scheme)만. 사용자가 못 바꿈.
- **개선**: 설정에서 라이트/다크/자동 토글(localStorage), 쿡모드 글자 크기 +/−. 주방에서 가독성.
- 노력: 낮음(CSS 변수 + 토글).

## Tier B — 실사용 편의 (노력 중간 · 효과 큼)

### B1. 장보기 슈퍼 코너별 그룹핑 ★추천
- **왜**: 지금 장보기는 이름+단위로 합산만. 마트에서 코너를 왔다갔다 하게 됨.
- **개선**: 재료를 **채소 · 육류/해산물 · 유제품/계란 · 두부/가공 · 통조림/건어물 · 면/쌀 · 조미료 · 기타**로 자동 분류해 동선대로 정렬. (nutrition 테이블·냉장고 그룹 로직 재활용)
- 노력: 중간.

### B2. 필터 확장 + 정렬
- **개선**: 카테고리/검색에서 **난이도 · 조리시간(≤15/30분) · 다이어트** 필터, **시간·난이도·별점·이름** 정렬. 지금은 하위분류만.
- 노력: 중간.

### B3. 검색 기록 + 자동완성
- **왜**: 사용자가 초기에 요청했던 항목(검색 이력). 현재 미구현.
- **개선**: 최근 검색어 저장/삭제, 인기 재료 칩 제안. (store에 searchHistory 키 추가)
- 노력: 낮음~중간.

### B4. 재료 대체 가이드 (한↔일) ★온테마
- **왜**: 일본에서 한국 재료 구하기 어려움. 앱 정체성과 정확히 맞음.
- **개선**: 대체 사전 페이지 + 레시피 내 배지. 예) 다시다→ほんだし/鶏がら, 국간장→うすくち醤油, 고춧가루 굵기, 멸치액젓→ナンプラー, 떡국떡/당면 구입처(업무슈퍼·카르디·돈키·한인마트).
- 노력: 중간(콘텐츠).

## Tier C — 큰 기능 (노력 높음)

### C1. 주간 식단 플래너
- 요일별로 레시피 배치 → 장보기 자동 합산 + 주간 영양 합계. cart 개념 확장.
- 노력: 높음.

### C2. 레시피 사진
- 이모지 대신 실제 사진이면 매력 급상승. **단 저작권** — 원문 사진 복사 불가. 본인 촬영 또는 AI 생성만. 이미지 수급·용량·지속관리 부담.
- 노력: 높음(지속 콘텐츠).

### C3. 콘텐츠 확장
- 베트남/인도/디저트 카테고리, 반찬 세트, 국물-반찬 밸런스, 명절·도시락 컬렉션.
- 노력: 중간~높음(지속).
